Submit to Web-CAT for automatic grading.

Web-CAT is an auto-grading system run out of Virginia Tech University. It allows you to submit Java projects. These projects are compiled and run against JUnit test cases like the ones you just wrote in Part 2.

Web-CAT provides two grades: one for the correctness of your code (whether it passes the test cases) and one for style (proper use of JavaDoc and indentation, etc). It will give you feedback each time you submit. You will have multiple submission attempts to improve your score, using that feedback as guidance.

You will be expected to submit all individual programming assignments usuing Web-CAT. It can be confusing at first, so that’s why we’re practicing today.

More Detail on the Correctness Score.

Look back at the screenshot in Part 3 of this lab, step 3c. You will see three different scores. All three of these contribute to the one Correctness/Testing score you get, which is out of 60 points and is shown at the top of the grading page (see screenshot in Part 3 of this lab, step 2d).

The first heading says Results from Running Your Tests (0%). This should match your results in NetBeans (or any other IDE you use) when you run your own test file locally. It runs the tests you wrote against your code.

The second heading says Code Coverage from YYour Tests (44%). This tells you how much of your code that the tests you wrote actually executes. In this lab, we only wrote two tests but there are five methods, so that is why your code coverage is around 2/5 or 40%. If you wrote tests for all of the methods, your code coverage would be 100%.

The third and final heading says Estimate of Problem Coverage (77%). This is similar to the first heading, except now it is testing your code agains the tests written by the instructional team for this course. If you do not have 100% here, there is something that the assignment specs asked you to do, and something that the instructional team is testing that your code does, but you did not pass that test. Usually, you will get the method name (sometimes corresponding to the test that failed), the exception that was thrown, and the line number in your code. Your overall Correctness/Testing score is the product of these three separate scores. So, in this specific case, the Correctness/Testing score comes from 0% × 44% × 77%, which is 0.

If you got 50% on your own tests, 75% on code coverage, and 45% on instructor-written tests, your overall correctness score would be 0.5 × 0.75 × 0.45 = 0.16875 or 16.875%. And 16.875% of 60 points is 0.16875 × 60 = 10.125 points out of 60. At the end of this lab , you may have had 100% on your own tests, 56% on code coverage, and 100% on instructor-written tests. This makes your overall Correctness/Testing score:

1.00 × 0.56 × 1.00 × 60 points = 33.6 points out of 60
